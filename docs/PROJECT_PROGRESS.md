# 项目开发成果总结

> 统计时间：2026-06-15  
> 当前阶段：企业试点候选版

## 1. 阶段结论

项目已经完成核心 MVP 和第二阶段主要能力建设，形成了从企业资料接入、异步解析、向量检索、可信问答到质量评估和 Docker 部署的完整闭环。

当前版本适合接入真实企业资料进行小范围试点，但尚不能视为正式生产版本。最新真实业务问答集的单轮结果已达到既定指标，仍需通过连续三轮质量门禁，并补齐可恢复删除、监控告警和试点运维流程。

## 2. 已完成能力

### 2.1 用户与知识库

- 邮箱注册、登录和 JWT 身份认证。
- 按用户隔离知识库、文档、会话和审计数据。
- 知识库创建、修改、删除和文档管理。
- 删除知识库时要求输入完整名称确认。
- 被删除的知识库文件先移动到服务器回收目录，降低误删后的恢复难度。

### 2.2 文档接入与解析

- 支持 PDF、DOCX、Markdown、TXT、CSV、HTML、PNG、JPEG 和 WebP。
- 支持批量上传、上传大小限制、格式与文件签名校验。
- 支持重复页眉页脚清洗、结构化切片和上下文重叠。
- 切片保存章节标题、页码、文本范围和内容类型元数据。
- 支持扫描 PDF OCR、PDF/DOCX/CSV 表格提取、文档内图片和独立图片理解。
- 使用持久化任务队列异步处理文档，支持失败重试和服务中断后的任务恢复。

### 2.3 RAG 问答与创作

- 使用 PostgreSQL、pgvector 和 FastEmbed 保存并检索语义向量。
- 混合语义相似度、关键词命中和章节相关性进行候选检索。
- 使用 Rerank、最低证据阈值和证据去重筛选最终上下文。
- 使用 LangGraph 编排检索、Rerank、证据判断、回答、事实校验和引用组装。
- 证据不足或校验失败时返回资料不足提示，降低编造风险。
- 引用包含来源文档、页码、相关度和命中位置附近的摘要。
- 支持 SSE 流式回答、最近对话上下文、会话历史、会话切换和删除。
- 提供产品卖点、营销文案、短视频脚本和 Prompt 参考四类创作模板。

### 2.4 工程与运维

- 使用 Alembic 管理数据库迁移。
- 使用 Docker Compose 部署前端、后端、Worker、PostgreSQL 和备份服务。
- 提供认证、上传和问答速率限制。
- 提供操作审计日志，且不记录密码、令牌、API 密钥和完整业务内容。
- 支持通过只读密钥文件向容器注入敏感配置。
- 定时生成 PostgreSQL 压缩备份并按保留周期清理。
- GitHub Actions 和本地脚本执行后端测试、Ruff 与前端生产构建。

## 4. 数据恢复与防误删改进

近期曾发生知识库删除操作，导致知识库记录、文档、切片和向量数据被级联删除。项目已使用删除前数据库备份恢复：

- 1 个知识库。
- 15 份文档。
- 107 个文档切片。
- 107 条向量数据。
- 对应会话与消息记录。

事故后已增加知识库名称确认、文件回收目录和恢复后即时备份。当前保护仍属于第一层防护：数据库记录仍会在删除时移除，下一阶段需要改为完整软删除，并提供可操作的回收站和恢复接口。

## 5. 当前边界

- 当前只支持用户级数据隔离，不支持组织、部门、角色、成员邀请和知识库共享。
- OCR 和图片理解依赖外部视觉模型，准确率、延迟和费用受模型服务影响。
- 复杂跨页表格重建、图表数值提取和多模态引用仍需要人工抽检。
- 文档 Worker 当前以单任务串行处理为主，不适合大规模并发导入。
- 审计日志已有查询 API，但尚无独立管理后台、指标看板和主动告警。
- 备份服务已可生成备份，但自动恢复验证和定期灾备演练尚未形成闭环。
- 回答仍可能存在事实遗漏或引用冗余，重要业务结论应结合原始资料人工确认。

## 6. 遗留风险与优先级

| 优先级 | 风险 | 下一步 |
|---|---|---|
| P0 | 质量指标只完成单轮达标 | 固化评估集并连续运行三轮门禁 |
| P0 | 数据库删除仍不可直接恢复 | 引入软删除、回收站和永久删除确认 |
| P1 | 缺少运行指标与主动告警 | 持久化问答摘要并增加诊断接口 |
| P1 | 备份可生成但恢复验证不足 | 自动校验备份并定期执行独立恢复演练 |
| P2 | 多模态和复杂表格质量覆盖不足 | 扩展 OCR、表格、图片评估案例 |
| P2 | 尚无多人协作权限 | 待企业试点反馈后再进入后续阶段 |
