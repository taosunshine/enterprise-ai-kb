# 下一阶段开发计划：企业试点稳定化

> 建议周期：4 周  
> 开发资源：1 人  
> 目标版本：企业试点版

## 1. 阶段目标

将当前“企业试点候选版”提升为可以稳定交付小范围企业试点的版本。本阶段优先解决回答质量波动、误删恢复、运行诊断和发布验收问题，不开发团队权限和高级 Agent 创作。

阶段完成时必须同时满足：

- 真实业务问答集连续运行 3 轮，检索命中率最低值 ≥ 90%。
- 引用准确率最低值 ≥ 80%，回答忠实度最低值 ≥ 80%。
- 各轮平均回答延迟均 ≤ 15 秒。
- 不可回答问题能够稳定拒答，不编造答案。
- 知识库和文档删除后可在 30 天内恢复。
- 备份能够在独立数据库中完成恢复验证。
- 关键失败能够通过诊断接口和日志定位。

## 2. 范围边界

### 本阶段包含

- RAG 质量稳定化与三轮质量门禁。
- 知识库和文档软删除、回收站、恢复与永久删除。
- 问答运行摘要、健康检查、质量摘要和失败任务诊断。
- 试点部署、备份恢复、验收和发布文档。

### 本阶段不包含

- 组织、部门、角色、成员邀请与共享权限。
- 高级 Agent、外部工具调用和复杂创作工作流。
- 大规模分布式任务队列和多节点部署。
- 完整商业化计费、内容审核和合规管理平台。

## 3. 第一周：质量基线与忠实度优化

### 开发内容

- 固化版本化真实业务评估集，记录数据集版本、文档版本、模型、Embedding、Rerank 和切片配置。
- 为评估集补充不可回答、多轮上下文、OCR、表格、图片和复杂对比问题。
- 优化证据阈值、重复引用过滤、引用与答案事实关联以及回答后事实校验。
- 对低分题输出失败类型：检索失败、引用错误、回答遗漏、回答超出证据或拒答错误。
- 每次完整评估连续运行 3 轮，并输出平均值、最低值、最高值和波动范围。

### 交付物

- 新版本评估集与对应测试资料。
- 三轮质量评估报告。
- 低分问题诊断清单与优化记录。

### 验收标准

- 检索命中率最低值 ≥ 90%。
- 引用准确率最低值 ≥ 80%。
- 回答忠实度最低值 ≥ 80%。
- 各轮平均回答延迟 ≤ 15 秒。
- 不可回答问题全部正确拒答。

## 4. 第二周：可恢复删除与数据安全

### 数据模型

为知识库和文档增加以下字段，并使用 Alembic 迁移：

| 字段 | 类型 | 说明 |
|---|---|---|
| `deleted_at` | nullable datetime | 非空表示已进入回收站 |
| `purge_after` | nullable datetime | 默认等于删除时间加 30 天 |
| `deleted_by_user_id` | nullable integer | 记录执行删除的用户 |

普通知识库和文档查询默认过滤 `deleted_at IS NULL`。进入回收站时保留数据库记录、切片、向量和文件；恢复时清空删除字段。只有永久删除操作才允许清理关联记录和文件。

### API 变更

| 接口 | 行为 |
|---|---|
| `DELETE /api/knowledge-bases/{id}?confirmation={name}` | 将知识库及文档移入回收站 |
| `DELETE /api/documents/{id}` | 将文档移入回收站 |
| `GET /api/trash` | 返回当前用户已删除的知识库和文档 |
| `POST /api/trash/{type}/{id}/restore` | 恢复指定知识库或文档 |
| `DELETE /api/trash/{type}/{id}?confirmation={name}` | 永久删除并清理关联数据与文件 |

`type` 只接受 `knowledge-base` 或 `document`。永久删除必须输入资源完整名称，并写入审计日志。后台清理任务每天处理超过 `purge_after` 的记录。

### 备份与恢复

- 备份生成后校验压缩文件可读取、SQL 内容非空并记录校验结果。
- 每周将最新备份恢复到独立临时数据库，执行表数量和关键记录抽样检查。
- 恢复演练不得覆盖当前运行数据库。

### 验收标准

- 删除知识库或文档后，普通列表不可见且回收站可见。
- 30 天内可以恢复数据库记录、文件、切片和向量。
- 错误确认名称无法执行永久删除。
- 自动清理只处理已超过保留期的软删除记录。
- 最新备份可以在独立数据库完成恢复验证。

## 5. 第三周：可观测性与诊断

### 问答运行摘要

新增问答运行摘要记录，保存：

- 用户、知识库、会话和请求标识。
- 检索候选数量、Rerank 后证据数量和最终引用数量。
- 检索、Rerank、回答、校验和总耗时。
- 是否拒答、拒答原因、事实校验是否通过和重试次数。
- 模型、Embedding、Rerank 和评估配置版本。

不得保存 API 密钥、密码、令牌、完整问题、完整回答或完整文档内容。问题只允许保存不可逆摘要或关闭记录。

### 诊断接口

| 接口 | 说明 |
|---|---|
| `GET /health` | 进程存活检查，保持现有行为 |
| `GET /ready` | 检查数据库、迁移状态、Worker 活跃度和必要模型配置 |
| `GET /api/quality/summary` | 返回当前用户知识库的近期质量与延迟摘要 |
| `GET /api/processing-tasks?status=failed` | 返回当前用户可见的失败任务与最后错误 |

前端增加失败任务提示、回收站入口和简洁质量状态，不建设独立管理后台。

### 验收标准

- 单次问答可通过请求标识定位各阶段耗时和失败原因。
- 数据库或 Worker 不可用时，`/ready` 返回非成功状态和明确原因。
- 用户只能查看自己的质量摘要、失败任务和回收站数据。
- 日志和数据库抽检确认不包含敏感密钥与完整业务内容。

## 6. 第四周：试点验收与发布

### 回归与验收

- 回归登录、上传、异步解析、OCR、表格、图片理解、问答、SSE、历史记录、删除、恢复和备份。
- 使用锁定版本的真实业务集完成连续三轮质量门禁。
- 人工抽检数字回答、复杂对比、跨文档问题、多模态问题和不可回答问题。
- 在全新环境执行一次 Docker Compose 部署和 Alembic 升级。
- 在独立数据库执行一次完整备份恢复。

### 发布文档

- 试点部署与升级说明。
- 日常检查、故障诊断和失败任务处理说明。
- 数据删除、回收站和永久删除说明。
- 数据库备份、恢复与灾备演练说明。
- 试点验收报告、版本发布记录和已知限制。

### 发布门槛

- 三轮质量门禁全部通过。
- 自动化测试、Ruff 和前端生产构建全部通过。
- 软删除恢复、永久删除确认和备份恢复完成验收。
- 无 P0 缺陷，P1 缺陷有明确规避方案和修复计划。

## 7. 测试计划

### 单元测试

- 软删除查询过滤、恢复、保留期和自动清理。
- 永久删除名称确认、用户隔离和审计记录。
- 证据阈值、引用去重、拒答和事实校验。
- 问答运行摘要脱敏和阶段耗时记录。
- 备份校验和恢复检查结果解析。

### 集成测试

- 删除知识库后回收站可见，恢复后文档、切片和向量重新可用。
- 文档软删除和恢复不影响同知识库中的其他文档。
- `/ready` 在数据库或 Worker 异常时返回正确状态。
- 普通问答和 SSE 问答均生成一致的运行摘要。
- 不同用户无法访问彼此的回收站、质量摘要和失败任务。

### 回归测试

- 现有认证、知识库、上传、问答、引用、历史记录和创作模板保持正常。
- 完整质量门禁连续运行 3 轮。
- Docker Compose 全新部署、升级和备份恢复通过。

## 8. 实施约束与默认值

- 由一人开发，优先使用当前 FastAPI、SQLAlchemy、PostgreSQL、pgvector、LangGraph 和 React 架构。
- 回收站默认保留 30 天，通过配置项调整。
- 团队权限和高级 Agent 在试点稳定化完成后再评估。
- 所有数据库结构变化必须通过 Alembic 迁移，不允许依赖手工建表。
- 新增接口必须保持用户数据隔离，并记录必要审计事件。
- 文档和发布说明只能陈述已验证能力，不将单轮达标描述为正式阶段验收通过。
